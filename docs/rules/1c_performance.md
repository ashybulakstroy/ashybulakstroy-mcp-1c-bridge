# Производительность 1С

## Базовые правила

- Для чтения использовать OData там, где достаточно CRUD/запроса справочников и регистров.
- Для бизнес-действий использовать RPC/HTTP-сервис 1С.
- Кэшировать справочники с низкой частотой изменений: контрагенты, номенклатура, склады.
- Не делать массовый поиск по строке на каждую позицию документа, если можно заранее получить кандидатов пачкой.
- Добавлять таймауты и retry/backoff для сетевых вызовов.
- Логировать длительность вызова, endpoint, тип операции и нормализованную ошибку.

## Красные флаги

- Более 100 строк в документе — предупреждение.
- Более 1000 строк — операция должна требовать отдельного подтверждения или пакетной обработки.
- Нет фильтра по периоду для регистров — предупреждение.
- Нет проверки остатков перед реализацией — ошибка бизнес-валидации.
