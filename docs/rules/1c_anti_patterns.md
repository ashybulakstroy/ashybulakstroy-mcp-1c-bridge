# Антипаттерны 1С

Эти правила применяются к разработке обработок 1С, HTTP-сервисов и MCP-инструментов AshybulakStroy MCP 1C Bridge.

## Запрещённые или рискованные практики

- Запросы к 1С в цикле, если можно выполнить один пакетный запрос.
- Выборка больших наборов без фильтра, периода или ограничения.
- Работа с документами записи без предварительной валидации.
- Проведение документа напрямую после AI-нормализации без шага `validate`.
- Неявное создание контрагентов, номенклатуры или складов без подтверждения пользователя.
- Игнорирование неоднозначных совпадений при поиске по наименованию.
- Возврат сырых ошибок 1С пользователю без нормализации.
- Жёстко зашитые имена объектов 1С без проверки через `$metadata`.

## Обязательное правило

Любая операция записи должна проходить pipeline:

```text
normalize -> validate -> review -> create -> validate_document -> post
```
